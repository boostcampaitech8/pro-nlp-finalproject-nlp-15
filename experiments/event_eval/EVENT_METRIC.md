# 이벤트 추출 성능 평가 (Event Extraction Evaluation)

뉴스 기사(`data/articles/*.csv`)와 추출된 이벤트 결과(`data/events/*.jsonl`)를 매칭하여 추출 품질을 정량적으로 평가하기 위한 지표 정의 및 실행 방법을 기술한다.

---

## 1. 평가 지표 개요

이벤트 추출 모델의 성능을 **일관성, 응집도, 분리도, 파편화, 오병합**의 5가지 관점에서 다각도로 분석한다.

| 지표명 | 영문명 | 정의 | 판단 기준 |
| :--- | :--- | :--- | :--- |
| **주제 일관성** | **Coherence ($C_V$)** | 이벤트 상위 단어 간의 의미적 일관성 | 높을수록 우수 (0.6↑ 양호) |
| **내부 응집도** | **Intra-event Similarity** | 동일 이벤트 내 기사 간 유사도 평균 | 높을수록 우수 |
| **외부 분리도** | **Inter-event Similarity** | 서로 다른 이벤트(Centroid) 간 유사도 | 낮을수록 우수 |
| **단순 파편화** | **Frag. (Singleton)** | 단일 기사로 구성된 이벤트의 비율 | 낮을수록 우수 |
| **유사 파편화** | **Frag. by Similarity** | 타 이벤트와 유사도가 높은 이벤트 비율 | 낮을수록 우수 (중복 방지) |
| **병합 오류율** | **Merging Error Rate** | 이질적인 기사가 섞인 것으로 의심되는 비율 | 낮을수록 우수 |

---

## 2. 지표별 산출 방식

### 2.1. 주제 일관성 ($C_V$)
이벤트별 빈도 상위 $k$개(기본 15개) 단어를 추출하여 임베딩을 수행한다. 각 단어 벡터와 이벤트 전체의 평균 벡터(Centroid) 간 코사인 유사도를 계산하여 평균한다. 
* **목적**: 추출된 이벤트 키워드들이 하나의 일관된 주제를 형성하는지 평가한다.

### 2.2. 내부 응집도 (Intra-event Similarity)
동일 이벤트에 속한 기사들 간의 모든 쌍(Pairwise)에 대한 코사인 유사도 평균을 산출한다. 기사가 1개인 싱글톤 이벤트의 경우 유사도를 1.0으로 처리한다.

### 2.3. 외부 분리도 (Inter-event Similarity)
각 이벤트의 Centroid를 산출한 후, 특정 이벤트 $i$와 이를 제외한 타 이벤트 Centroid 간의 코사인 유사도 평균을 구한다. 이 값이 낮을수록 이벤트 간의 독립성이 확보된 것으로 간주한다.

### 2.4. 파편화 분석 (Fragmentation)
하나의 사건이 여러 개의 이벤트로 과잉 분할되었는지 평가한다.
* **Singleton Rate**: 전체 이벤트 중 기사 수가 1개인 이벤트의 비중을 산출한다.
* **Similarity-based**: 타 이벤트와의 Centroid 유사도가 임계치(기본 0.85) 이상인 이벤트의 비율을 산출하여 중복 추출 여부를 판단한다.

### 2.5. 병합 오류율 (Merging Error Rate)
내부 응집도(Intra-event similarity)가 설정된 임계치(기본 0.4) 미만인 이벤트의 비율을 산출한다. 이는 서로 다른 주제가 하나의 이벤트로 오병합되었을 가능성을 나타내는 지표다.

## 3. 실행 가이드

### 3.1. 설정 관리 
`experiments/event_eval/event_evaluate.yaml` 파일에서 주요 파라미터를 조정할 수 있다.
* `embedding_model`: 문장 및 단어 임베딩에 사용할 모델
* `merging_threshold`: 오병합 판정 임계치
* `fragmentation_similarity_threshold`: 파편화 판정 유사도 기준

### 3.2. 실행 명령어
프로젝트 루트 디렉토리에서 아래 명령어를 수행한다.
```bash
python -m experiments.event_eval.event_evaluation
